資料整備流程

資料整備任務類型共分為:

  • 機器學習 (machine_learning_task)
  • 規則模型 (rule_task)
機器學習模型資料

建立任務請選擇 機器學習模型資料

請至少準備以下類型資料集:

  • 訓練資料(train)
  • 驗證資料(dev)
  • 測試資料(test)

上傳資料欄位，*為必要欄位:

  • title
  • author
  • s_id
  • s_area_id
  • *content
  • post_time
  • *label
  • *document_type
    • train
    • dev
    • test
    • ext_test
規則模型資料

建立任務請選擇 規則模型資料

上傳資料欄位，所有欄位均為必要欄位:

  • content
  • label
  • rule_type
    • keyword
    • regex
  • match_type
    • start
    • end
    • partially
    • exactly
